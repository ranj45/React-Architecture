// Layer Responsibilities

üèóÔ∏è

Config Layer bottom

Controls how your app is built and run. Everything else depends on this layer.

vite.config.js ‚Üí build settings, proxy, aliases .env.* ‚Üí environment variables package.json ‚Üí scripts, dependencies index.html ‚Üí HTML shell, title, meta tags
changed rarelyaffects everything
‚ö°

Boot Layer app start

Runs once at startup. Sets up providers, global state, routing.

main.jsx ‚Üí ReactDOM.createRoot ‚Üí wraps with <BrowserRouter> ‚Üí wraps with <Provider store> ‚Üí wraps with <AuthProvider> App.jsx ‚Üí defines all routes ‚Üí handles layouts
setup onceproviders here
üîå

Data Layer services

All external communication. Components never call fetch() directly.

services/api.js ‚Üí axios base config services/auth.js ‚Üí login, logout, tokens services/users.js ‚Üí user CRUD calls // Uses: import.meta.env.VITE_API_URL
no UI herepure functions
üé£

Logic Layer hooks

Custom hooks bridge data layer to UI. Reusable stateful logic.

hooks/useAuth.js ‚Üí auth state + methods hooks/useFetch.js ‚Üí generic data fetcher hooks/useForm.js ‚Üí form state + validation hooks/useLocalStorage ‚Üí persist state // Calls services, returns state to components
no JSXreusable
üìë

Page Layer pages/

Route-level components. Compose smaller pieces. One page per route.

pages/Home/index.jsx pages/Dashboard/index.jsx pages/Profile/index.jsx // Pages: // - Use hooks for data // - Compose components // - Handle loading/error states // - NO direct API calls
tied to routessmart components

UI Layer components/

Dumb, reusable pieces. Get data via props. Know nothing about API or routing.

components/ui/ ‚Üí Button, Input, Modal components/layout/ ‚Üí Header, Footer, Sidebar components/forms/ ‚Üí LoginForm, SignupForm // Rule: if it uses useNavigate // or calls an API, it's a page, not a component

// What's Safe to Put in VITE_ Variables

‚úÖ

Safe ‚Äî put in VITE_

These are designed to be public or are URLs, not secrets.

VITE_STRIPE_PUBLIC_KEY=pk_live_... ‚úÖ public by design VITE_GOOGLE_MAPS_KEY=AIza... ‚úÖ restrict in console VITE_FIREBASE_API_KEY=... ‚úÖ Firebase rules protect VITE_API_URL=https://api.myapp.com ‚úÖ just a URL VITE_APP_NAME=MyApp ‚úÖ not sensitive VITE_AUTH0_CLIENT_ID=abc123 ‚úÖ public Auth0 ID
‚ùå

Never ‚Äî keep without VITE_

These would be visible to anyone who opens browser DevTools.

VITE_DATABASE_URL=postgres://... ‚ùå server only VITE_JWT_SECRET=mysecret ‚ùå signing secret VITE_AWS_SECRET_KEY=abc123 ‚ùå full access key VITE_STRIPE_SECRET=sk_live_... ‚ùå server-side only VITE_SENDGRID_API_KEY=SG.xxx ‚ùå email server key # These should go in your BACKEND .env # Never in your React frontend

// vite.config.js ‚Äî Every Section Explained

üîå

plugins required

Extend Vite's capabilities. Add here when you need new file support.

plugins: [ react(), // JSX + Fast Refresh svgr(), // import Logo from './logo.svg' viteImagemin(), // compress images VitePWA(), // make app installable eslintPlugin(), // lint on save ] // Change when: new file type, PWA, bundle analysis
üìÅ

resolve.alias must have

Clean imports. Set once, use everywhere. Saves hours of fixing relative paths.

resolve: { alias: { '@': path.resolve('./src'), '@comp': path.resolve('./src/components'), '@hooks': path.resolve('./src/hooks'), '@utils': path.resolve('./src/utils'), } } // Before: '../../../../utils/format' // After: '@utils/format'
üñ•Ô∏è

server dev only

Dev server settings. The proxy is critical to avoid CORS errors locally.

server: { port: 3000, open: true, proxy: { '/api': { target: 'http://localhost:8080', changeOrigin: true, // /api/users ‚Üí localhost:8080/api/users }, '/auth': { target: 'http://localhost:8080', } } }
üì¶

build production

Controls your production bundle. Critical for performance and caching.

build: { outDir: 'dist', sourcemap: true, rollupOptions: { output: { manualChunks: { // separate vendor cache vendor: ['react', 'react-dom'], router: ['react-router-dom'], ui: ['@radix-ui/react-dialog'], } } } }
üé®

css styling

CSS Modules configuration and preprocessor setup.

css: { modules: { localsConvention: 'camelCase' // styles.myClass instead of styles['my-class'] }, preprocessorOptions: { scss: { additionalData: `@import "./src/styles/vars";` // vars available in every .scss file } } }

Access env vars inside vite.config.js itself using loadEnv.

import { defineConfig, loadEnv } from 'vite' export default defineConfig(({ mode }) => { // import.meta.env doesn't exist in config // use loadEnv instead: const env = loadEnv(mode, process.cwd(), '') return { server: { proxy: { '/api': { target: env.VITE_API_URL } } } } })

// Quick Reference ‚Äî What to Change When

üîÄ

New Feature / Page

ALWAYS touch: pages/FeatureName/index.jsx ‚Üê create App.jsx ‚Üê add route constants/routes.js ‚Üê add path constant MAYBE touch: components/layout/Header ‚Üê nav link store/slices/featureSlice.js ‚Üê if needs state services/featureService.js ‚Üê if needs API hooks/useFeature.js ‚Üê if needs logic
üåê

API Change

ALWAYS touch: services/[name]Service.js ‚Üê endpoint change hooks/use[Name].js ‚Üê if response shape changed MAYBE touch: .env.development ‚Üê new URL .env.production ‚Üê new URL .env.example ‚Üê document new var vite.config.js ‚Üê new proxy rule constants/api.js ‚Üê new endpoint string
üé®

UI Redesign

ALWAYS touch: components/ui/[component] ‚Üê changed component *.module.css ‚Üê scoped styles MAYBE touch: styles/globals.css ‚Üê global changes styles/variables.css ‚Üê new design tokens index.html ‚Üê new font import vite.config.js ‚Üê new font plugin
üîê

Auth Change

ALWAYS touch: services/authService.js ‚Üê auth logic context/AuthContext.jsx ‚Üê auth state hooks/useAuth.js ‚Üê hook update MAYBE touch: .env files ‚Üê new auth keys main.jsx ‚Üê new provider App.jsx ‚Üê route protection components/PrivateRoute.jsx ‚Üê guard logic
‚ö°

Performance Issue

ALWAYS touch: vite.config.js ‚Üê manualChunks App.jsx ‚Üê React.lazy routes MAYBE touch: heavy components ‚Üê dynamic import() images ‚Üí public/ ‚Üê move large assets hooks/useFetch.js ‚Üê add caching store/ ‚Üê memoize selectors
üöÄ

New Deployment Target

ALWAYS touch: .env.[mode] ‚Üê create env file package.json ‚Üê add build script MAYBE touch: vite.config.js ‚Üê base path, output .gitignore ‚Üê new env files CI/CD config ‚Üê new pipeline step .env.example ‚Üê document new vars

New npm Package

ALWAYS touch: package.json ‚Üê auto (npm install) package-lock.json ‚Üê auto (npm install) MAYBE touch: vite.config.js ‚Üê if plugin needed main.jsx ‚Üê if provider needed constants/ ‚Üê if config values .env files ‚Üê if API keys needed vite-env.d.ts ‚Üê TypeScript types

// The 3 Questions to Ask Before Any Change

1. Does this change affect how the app boots? Yes ‚Üí touch main.jsx, possibly vite.config.js 2. Does this change affect where data comes from? Yes ‚Üí touch services/, hooks/, .env files 3. Does this change affect what the user sees? New screen ‚Üí touch pages/, App.jsx, constants/routes.js Same screen ‚Üí touch components/, *.module.css, styles/
